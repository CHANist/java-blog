Necessity in Developing Type Conversion#

In Spring, there are lots of scenarios where it is needed to change type from A to B. For example,

  1. When using AnnotationConfigApplicationContext, we normally inject value stored in properties file to @Component using @Value annotation.

    @Servicepublic class ConnectionService {    @Value("${connection.urls}")    private List<String> urls; // url list to try, in case one cannot be connected    @Value("${connection.maxtrial}")    private int maxTrial; // max trial to try before throwing an exception}

    And the corresponding properties file 

    connection.urls=http://192.168.100.1:3000,http://192.168.100.2:3000connection.maxtrial = 3

  1. For XML Equivalent, 

    public class ConnectionService {    private List<String> urls; // url list to try, in case one cannot be connected    private int maxTrial; // max trial to try before throwing an exception        // ...}

    With the XML Configuration File being 

    <?xml version="1.0" encoding="UTF-8"?><beans xmlns="http://www.springframework.org/schema/beans"       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"       xsi:schemaLocation="http://www.springframework.org/schema/beans        https://www.springframework.org/schema/beans/spring-beans.xsd">
    <bean id="connectionService" class="com.thechanist.tuling.spring.ch03.services.ConnectionService">        <property name="urls" value="http://192.168.100.1:3000,http://192.168.100.2:3000" />        <property name="maxTrial" value="3" />    </bean></beans>

In both cases, Spring needs to automatically convert type so that the property can be injected in a proper way. But notice that I am using , as a delimiter to separate a String, and definitely others may have other choice to do so. So how can Spring know how to do the conversion?

Mechanism Supported#

Spring actually support multiple ways for end user to register the conversion. Let's have a look at each of them.

Instead of specifying the mechanism on how it works, I will provide an example here instead.

PropertyEditor#

This is a class injava.beans package, which is inside JDK.

First create a CustomPropertyEditor that implements PropertyEditor, which is the core code for conversion. 

public class StringToListPropertyEditor extends PropertyEditorSupport implements PropertyEditor {  @Override  public void setAsText(String text) throws IllegalArgumentException {    List<String> list = Arrays.asList(text.split(","));    this.setValue(list);  }}

As this is a Mechanism supplied by JDK, we can definitely test if the property editor is working as expected without using Spring. 

@Testpublic void test() {    StringToListPropertyEditor propertyEditor = new StringToListPropertyEditor();    propertyEditor.setAsText("apple,banana");    List<String> fruits = (List) propertyEditor.getValue();    assertEquals(2, fruits.size());    assertEquals("apple", fruits.get(0));    assertEquals("banana", fruits.get(1));}
note

At this stage, we have only create a custom PropertyEditor , however we still miss the following

  • We have not tell Spring to use the PropertyEditor. We need some way to register it in Spring
  • Spring may need to support many PropertyEditors, but how can Spring determine which one to use? We should support the return setValue type, e.g. in above case List to Spring so that Spring can better identify whether such PropertyEditor can be used for the conversion or not.

To register the PropertyEditor, we can do either of the following:

  1. Creating customEditorConfigurer bean using @Bean.

    @Configurationpublic class AppConfig {    @Bean    public CustomEditorConfigurer customEditorConfigurer() {      CustomEditorConfigurer customEditorConfigurer = new CustomEditorConfigurer();      Map<Class<?>, Class<? extends PropertyEditor>> propertyEditorMap = new HashMap<>();
      // so that when @Value is placed on a field with type List, Spring will try to use the propertyEditor for conversion      propertyEditorMap.put(List.class, StringToListPropertyEditor.class);      customEditorConfigurer.setCustomEditors(propertyEditorMap);      return customEditorConfigurer;    }}
    caution

    Do the beanName of CustomEditorConfigurer must be customEditorConfigurer, in order for the above to work properly? Why or why not?

  2. Create a custom PropertyEditorRegistrar, which I think is more user friendly when using it with XML Configuration. As specifying a map in XML format is way more complicated setup. 

    public class AppPropertyEditorRegistrar implements PropertyEditorRegistrar {    @Override    public void registerCustomEditors(PropertyEditorRegistry registry) {\        registry.registerCustomEditor(List.class, new StringToListPropertyEditor());    }}
    <?xml version="1.0" encoding="UTF-8"?><beans xmlns="http://www.springframework.org/schema/beans"       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"       xsi:schemaLocation="http://www.springframework.org/schema/beans        https://www.springframework.org/schema/beans/spring-beans.xsd">
    <bean id="customEditorConfigurer" class="org.springframework.beans.factory.config.CustomEditorConfigurer">        <property name="propertyEditorRegistrars">            <list>                <bean class="com.thechanist.tuling.spring.ch03.conversion.registrar.AppPropertyEditorRegistrar" />            </list>        </property>    </bean>        <bean id="connectionService" class="com.thechanist.tuling.spring.ch03.services.ConnectionService">        <property name="urls" value="http://192.168.100.1:3000,http://192.168.100.2:3000" />        <property name="maxTrial" value="3" />    </bean></beans>

    Here is the test code for the above setup. 

    @Testpublic void xmlClassTest() {    ClassPathXmlApplicationContext applicationContext = new ClassPathXmlApplicationContext("spring.xml");    ConnectionService connectionService = (ConnectionService) applicationContext.getBean("connectionService");
    List<String> urls = connectionService.getUrls();    assertEquals(2, urls.size());    assertEquals("http://192.168.100.1:3000", urls.get(0));    assertEquals("http://192.168.100.2:3000", urls.get(1));}

ConversionService#

This is a service Spring provides to allow greater conversion capability.

Same as PropertyEditor, we need to specify the conversion and when the conversion should take effectively. 

@Componentpublic class StringToListConverter implements GenericConverter {    @Override    public Set<ConvertiblePair> getConvertibleTypes() {        return Collections.singleton(new ConvertiblePair(String.class, List.class));    }
    @Override    public Object convert(Object source, TypeDescriptor sourceType, TypeDescriptor targetType) {        String value = (String) source;        String[] values = value.split(",");        return Arrays.asList(values);    }}

and then tell Spring to use the Converter

@Configuration@PropertySource("classpath:app.properties")public class AppConfig {    @Autowired    private List<GenericConverter> genericConverters;        @Bean    public ConversionServiceFactoryBean conversionService() {        ConversionServiceFactoryBean serviceFactoryBean = new ConversionServiceFactoryBean();        serviceFactoryBean.setConverters(new HashSet<>(genericConverters)));        return serviceFactoryBean;    }    }
